# OneEstimate 要件定義書

**作成日:** 2026年3月1日
**バージョン:** 5.0
**ステータス:** ドラフト

---

## 1. プロジェクト概要

### 1.1 目的

工務店向けの住宅見積もりシミュレーションアプリケーション「OneEstimate」を開発・リリースする。
既存の demo-app（静的 HTML/JS によるプロトタイプ）の機能を、one-estimate（Next.js 基盤）上に再構築し、実用的な業務アプリケーションとして提供する。

### 1.2 背景

- **demo-app の現状:** Vanilla JS + LocalStorage のみで動作するクライアントサイド完結型のプロトタイプ。工務店の商談時に住宅シリーズ選択→見積もり→資金計画書作成までをカバーするが、データの永続化・マルチデバイス対応ができない。
- **one-estimate の現状:** Next.js 14 + Supabase 認証 + Prisma ORM + Neon DB（PostgreSQL）の基盤が構築済み。ビジネスロジックは未実装。

### 1.3 ターゲットユーザー

| ユーザー種別 | 説明 |
|---|---|
| **工務店管理者** | シリーズ・オプション・価格などのマスターデータを管理する工務店の経営者・管理者 |
| **工務店営業担当** | 商談時に見積もりシミュレーションを操作し、見積書・資金計画書を作成する営業スタッフ |

### 1.4 マルチテナント・権限設計

#### 1.4.1 テナント構成

- **1工務店 = 複数ユーザー** の構成とする
- Company（工務店）テーブルを設け、User / Customer / Estimate / CompanyInfo / 各マスターデータに `companyId` を持たせる
- 同じ工務店に所属するユーザーは顧客データ・見積もりデータ・マスターデータを共有する
- 異なる工務店のデータは完全に分離する

#### 1.4.2 ロール・権限モデル

| ロール | /admin | 見積もりシミュレーション | 顧客管理 | 会社情報 | マスターデータ |
|---|---|---|---|---|---|
| **管理者（ADMIN）** | アクセス可 | 利用可 | 全顧客閲覧可 | 編集可 | CRUD可 |
| **営業担当（SALES）** | アクセス不可 | 利用可 | 同じ工務店の全顧客閲覧可 | 閲覧のみ | 閲覧のみ |

- User テーブルに `role` フィールド（`ADMIN` / `SALES`）を追加
- 工務店の最初のユーザー（サインアップ時）は自動的に `ADMIN` ロールとなる
- 管理者が営業担当を招待・ロール変更できる
- 営業担当が `/admin` 配下にアクセスした場合はダッシュボードにリダイレクト

### 1.5 システム形態

- Web アプリケーション（Next.js）
- ローカル環境でも本番環境と同等に動作する構成（Neon DB 共通）
- Web ブラウザで動作（PC / タブレット対応）
- 課金システムなし（認証のみ）
- マルチテナント対応（1工務店=複数ユーザー）

### 2.3 見積もり計算ロジック

```
■ 見積書 = A基本本体価格 + B付帯工事費 + Cオプション工事費 + Dその他諸費用

■ 原価+粗利益率 → 販売価格モデル:
  基本本体価格:
    坪数別実行予算 = 基準実行予算(30坪基準) × 坪数係数
    販売価格 = 坪数別実行予算 ÷ (1 − 粗利益率)
    ※販売価格は手動上書き可
    坪数係数: 15坪=0.615 ～ 45坪=1.384（30坪=1.000、31段階）

  変動費・オプション:
    各項目に原価と販売価格差額を設定
    販売価格差額 = 原価ベースから粗利益率で自動算出（手動上書き可）

A = 基本本体価格（販売価格）
B = 付帯工事費（工務店が初期設定で入力する概算金額）
C = 変更工事費（変動費合計）+ 住設オプション（差額合計）+ 照明・カーテン等（工務店設定）
D = その他諸費用（工務店が初期設定で入力する概算金額）

資金計画: 月々返済額 = PMT関数(借入金額, 年利, 返済期間)
```

## 7. 非機能要件

### 7.1 ローカル/本番 共通構成

| 項目 | 方針 |
|---|---|
| DB | **Neon DB（PostgreSQL）をローカル/本番共通で使用。** ローカルからクラウドDBに直接接続する。開発/本番の分離は Neon Branching で行う |
| 画像 | ローカル: public/uploads/、本番: Supabase Storage |
| 認証 | Supabase クラウドプロジェクトにローカルからも接続 |
| OpenAI | ローカル/本番共通: 環境変数 `OPENAI_API_KEY` |
| 環境切替 | `.env.local` / `.env.production` による `DATABASE_URL` 等の切替のみ |

**Neon Branching の活用:**
- `main` ブランチ = 本番DB
- `dev` ブランチ = 開発用DB（本番のスキーマ+データをコピーして瞬時に作成可能）
- ローカル開発では `dev` ブランチの接続URLを `.env.local` に設定
- スキーマ変更のテスト後、本番ブランチにマイグレーションを適用

|---|---|
| ページ表示速度 | 初回表示 3秒以内 |
| 見積もり計算 | リアルタイム（入力変更後100ms以内に反映） |
| AI推薦生成 | 5秒以内（OpenAI API レスポンス） |
| Excel出力 | 5秒以内 |

### 7.5 エラーハンドリング方針

| エラー種別 | ユーザーへの通知 | 詳細ログ |

## 10. 帳票項目定義

### 10.0 帳票設計方針

#### 10.0.1 カテゴリ統一原則

見積書と資金計画書の項目カテゴリは以下の4分類で統一する。カテゴリがズレると工務店の運用が混乱するため、両帳票で同じ分類体系を使う。

| カテゴリ | 見積書での呼称 | 資金計画書での呼称 |
|---|---|---|
| A | 基本本体価格 | 建物工事費 |
| B | 付帯工事費 | 付帯工事費 |
| C | オプション工事費 | 変動工事・オプション工事 |
| D | その他諸費用 | その他諸経費 |

資金計画書はこの4分類に加えて、E（事務手数料）・F（土地費用）・G（補助金等）を持つ。

#### 10.0.2 UI と出力の分離

- **UI（アプリ画面）:** 現在のステップ形式（シリーズ選択→アンケート→オプション選択→見積→資金計画）を維持
- **出力（帳票）:** 工務店の実務で使える"定番"の書式に寄せる。UI上の入力ステップと帳票上のカテゴリ分類は異なってよい

#### 10.0.3 動的金額と固定金額の分離

帳票に表示される金額は、データソースにより2種類に分類される。

| 分類 | 説明 | 例 |
|---|---|---|
| **動的金額** | アプリでユーザーが選択・入力した結果が自動反映される | 基本本体価格（坪数選択）、変更工事費（形状・部屋数等）、住設オプション（グレード選択） |
| **固定金額** | 工務店ごとに常にかかる費用。管理画面の初期設定で入力し、見積書・資金計画書に反映 | 付帯工事費（屋外給排水等）、その他諸費用（地鎮祭等）、事務手数料（登記・保険等） |

#### 10.0.4 項目の表示/非表示管理

- 各セクション（B〜G）の項目には**表示/非表示のトグル**を設ける
- 工務店が使わない項目は非表示にできる（例: 地盤改良が不要な案件では非表示）
- 項目数は各セクションの最大数まで追加可能とし、運用上の制限は管理画面で設定
- 非表示項目は帳票出力（見積書・資金計画書）には含めない
- 顧客ごとの見積もり作成時にも、個別に項目の表示/非表示を切替可能とする

**月々返済額の計算（PMT関数）:**
```
借入額 = 費用総額 − 自己資金
月利 = 年利 / 100 / 12
返済回数 = 返済期間 × 12
月々返済額 = 借入額 × 月利 × (1 + 月利)^返済回数 / ((1 + 月利)^返済回数 - 1)
※年利が0の場合: 借入額 / 返済回数
年間返済額 = 月々返済額 × 12 + ボーナス返済 × 2
返済総額 = 年間返済額 × 返済期間(年)
```

| B 付帯工事費 | 税抜小計 → ×0.1 四捨五入 → 税込 |
| C 変動工事・オプション | 税抜小計 → ×0.1 四捨五入 → 税込 |
| D その他諸経費 | 税抜小計 → ×0.1 四捨五入 → 税込 |
| E 事務手数料 | 税抜小計 → ×0.1 四捨五入 → 税込 |
| F 土地費用 | 非課税（消費税なし） |
| G 補助金等 | 消費税なし（マイナス計上） |

#### 10.2.14 見積もり→資金計画書の自動連携ルール

| 連携先 | データソース | 連動ロジック |
|---|---|---|
| A > 本体工事費用 | 基本本体価格の販売価格 | 坪数選択 → 販売価格を自動反映 |
| C > 変更工事費 | 変動費の各選択差額 | 形状+部屋数+吹抜+屋根+外部空間の差額合計 |
| C > 住設オプショングレードアップ | オプションの全12カテゴリ差額 | キッチン〜フローリングの差額合計 |
| F > 土地購入費用 | アンケートQ4の土地予算 | Q4回答の土地予算がそのまま反映 |

## 11. 用語集

| 用語 | 説明 |
|---|---|
| シリーズ | 住宅の基本グレード（スタンダード/コンフォート/プレミアム/ラグジュアリー等） |
| 坪数 | 建築面積。1坪 ≒ 3.3㎡ |
| 坪数係数 | 坪数に応じた価格調整係数。30坪 = 1.000 を基準 |
| 変動費 | 間取り形状・部屋数・吹き抜け・屋根形状・外部空間による追加/減額費用 |
| オプション | 標準仕様からの変更・追加設備（キッチン、断熱、外壁等） |
| 資金計画書 | 建築総費用・ローン・補助金・ランニングコストを含む総合資金計画シート |
| PMT関数 | ローンの月々返済額を算出する関数 |
| SpecLabel | シリーズの仕様項目名を動的に管理するマスター |
